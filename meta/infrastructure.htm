Infrastructure Resources 6.0

NameAliasesDescription
Support
Generally useful resources whereever resources are used
ListCollection, WorkingListA set of information summarized from a list of other resources.
MediaA photo, video, or audio recording acquired or used in healthcare. The actual content maybe inline or provided by direct reference.
OtherZ-resource, Extension-resourceOther is a conformant for handling resource concepts not yet defined for FHIR or outside HL7's scope of interest.
DocumentReferenceA reference to a document.
Binary(Not defined yet)
BinaryPure Binary Content - opaque to FHIR
Audit
Used to track when information was changed or who was responsible for it
ProvenanceProvenance information that describes the activity that lead to the creation of a set of resources. This information can be used to help determine their reliability or trace where the information in them came from. The focus of the provenance resource is record keeping, audit and traceability, and not explicit statements of clinical significance.
SecurityEventA record of an event made for purposes of maintaining a security log. Typical uses include detection of intrusion attempts and monitoring for inappropriate usage.
Exchange
Used to support the process of exchanging data
DocumentA documentation of healthcare-related information that is assembled together into a single statement of meaning that establishes its own context. A document is composed of a set of resources that include both human and computer readable portions. A human may attest to the accuracy of the human readable portion and may authenticate and/or sign the entire whole. A document may be kept as a set of logically linked resources, or they may be bundled together in an atom feed.
MessageThe root for a transmission either requesting or responding to an action. The resource(s) that are the subject of the action as well as other Information related to the action are typically transmitted in a bundle of which the Message resource instance is the first resource in the bundle.
OperationOutcomeA collection of error, warning or information messages that result from a system action.
QueryA description of a query with a set of parameters.
Conformance
Make statements about how resources are used
ConformanceA conformance statement about how an application or implementation supports FHIR or the set of requirements for a desired implementation.
ValueSetA value set specifies a set of codes drawn from one or more code systems.
ProfileTemplateA Resource Profile - a statement of use of one or more FHIR Resources. It may include constraints on Resources and Data Types, Terminology Binding Statements and Extension Definitions.

Additional Resources will be added in the future. A list of hypothesized list of resources can be found on the HL7 wiki. Feel free to add any you feel are missing or engage with one of the HL7 Work Groups to submit a proposal to define a resource of particular interest.
